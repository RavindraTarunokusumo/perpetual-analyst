# Specs

Feature implementation is driven by accepted specs in this directory.

Each active implementation session should identify one accepted spec path during
Workflow Step 1 and derive its `TODO.md` session entry from that spec. Specs in
this directory are the current authority for implementation scope; older
historical specs remain archived under `docs/superpowers/specs/`.

An accepted spec must include:

- status and date
- user-facing goal
- scope and non-goals
- architecture and file boundaries
- data model or migration impact
- validation plan
- unresolved questions, if any

Do not implement from a spec that has unresolved blocking questions.
